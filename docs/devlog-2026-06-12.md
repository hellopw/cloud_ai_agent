## 2026-06-12 开发记录

### 1. 聊天 SSE 事件攒批渲染

**问题**: Anthropic streaming API 每次 `text_delta` 只有 1-2 个字符，每个事件触发一次 React re-render，页面卡顿。

**方案**: 使用 `requestAnimationFrame` 批处理。

**改动**: `frontend/src/pages/ChatPage.tsx`
- 新增 `rafRef`、`needsFlushRef` 控制帧内去重
- `scheduleContentUpdate()` — 标记脏位，每帧最多调度一次 RAF
- `flushStreaming()` — RAF 回调，读取 `streamingRef.current` 写入 state
- `cancelPendingFlush()` — agent_end/message_end 时取消未完成 RAF 并直接 finalize

**效果**: 每帧最多一次渲染（~60fps），不论收到多少个 delta。

### 2. Dockerfile 模板外部化

**问题**: Dockerfile 模板硬编码在 Go 源码中，修改需重新编译。

**方案**: 提取到 `container-wrapper/dockerfiles/` 目录，运行时加载，内置常量作为 fallback。

**文件**:
| 文件 | 用途 |
|---|---|
| `container-wrapper/dockerfiles/pi.dockerfile` | PI agent 模板 |
| `container-wrapper/dockerfiles/claude-code.dockerfile` | Claude Code agent 模板 |
| `container-wrapper/dockerfiles/codex.dockerfile` | Codex agent 模板 |
| `container-wrapper/dockerfiles/team.dockerfile` | Team 模式模板 |
| `backend/internal/codegen/dockerfile.go` | 新增 `TemplatesDir`、`loadTemplate()`、`getTemplate()` |
| `backend/internal/codegen/dockerfile_types.go` | 保留 fallback 常量，`GenerateDockerfileByAgentType` 改为文件加载 |
| `backend/internal/codegen/team_dockerfile.go` | `GenerateTeamDockerfile` 改为 `getTemplate("team")` |
| `backend/internal/service/agent.go` | `NewAgentService` 中自动设置 `codegen.TemplatesDir` |

**加载优先级**: 外部文件 → 内置 fallback 常量

### 3. 聊天记录持久化

**问题**: 刷新页面后聊天记录丢失。

**方案**: 聊天消息持久化到 SQLite，前端加载/保存通过 REST API。

**后端**:

| 文件 | 改动 |
|---|---|
| `internal/model/models.go` | 新增 `ChatMessage` 结构体 |
| `internal/store/store.go` | 新增 `chat_messages` 表（关联 instances，CASCADE 删除） |
| `internal/store/chat_messages.go` | **新文件** — `CreateChatMessage`、`ListChatMessages` |
| `internal/api/handlers.go` | 新增 `listChatMessages`（GET）、`createChatMessage`（POST） |
| `internal/api/router.go` | 新增路由 `GET/POST /api/instances/{id}/messages` |

**API**:
```
GET  /api/instances/{id}/messages  → 返回该实例的所有消息（按时间排序）
POST /api/instances/{id}/messages  → 保存一条消息
  Body: { "role": "user|assistant|tool|system", "content": "...", "tool_call": "..." }
```

**前端**: `frontend/src/pages/ChatPage.tsx`
- 页面加载时 `GET` 拉取历史消息恢复展示
- 用户发送时 `saveMessage('user', ...)` 持久化
- tool_call / tool_result 到达时持久化
- agent_end / message_end 时将累积的完整回复持久化
- 流式 delta 不单独存储（只存最终结果）

### 4. 部署注意事项

- 服务器从 `frontend/` 目录读取静态文件（非 `static/`）
- 前端部署: `scp dist/* → ~/cloud_ai_agent/frontend/`
- Dockerfile 模板需同步部署: `scp dockerfiles/*.dockerfile → ~/cloud_ai_agent/container-wrapper/dockerfiles/`
- 编译: `GOOS=linux GOARCH=amd64 CGO_ENABLED=0 go build`
- 重启: `pkill cloud-ai-agent && PORT=29080 nohup ./cloud-ai-agent > logs/server.log 2>&1 &`
